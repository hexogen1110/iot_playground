5. Using west Commands to Build Your Project

After creating your project, use the following west commands to build and manage it in the terminal:

  • Build the Project:

    Replace <board_name> with the target board name (e.g., for m5stickC plus, I choose m5stickc_plus/esp32/procpu, and specify the module we need to build. For example, we use samples/hello_world as our first step).
    -p for –pristine, always clean before build

    Note:
    During the first boot, Zephyr RTOS reports that the chip revision 1 used in the M5StickC Plus is outdated. To support this older hardware version, you need to enable CONFIG_ESP32_USE_UNSUPPORTED_REVISION=y in the prj.conf file

  • Clean the Build Directory:

    This command cleans the build directory before starting a fresh build.

  • Flash the Built Firmware to the Target Board:

    This requires the target board to be connected to your computer. For ESP boards, set the baud rate to 115200, as there is a known issue in some forums where the west flash command may fail at lower baud rates.

  • Run a Debug Session:

    This starts a debug session if the board and debugger are properly configured.

  • Open a Serial Console:

    Use this to interact with the board’s serial interface.
    Connect the device to USB then you will see the boot log from serial outpout:
